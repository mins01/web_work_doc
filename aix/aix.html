퀵

  • 현재 메모리 사용율은?
    • topas에서 % COMP 부분 참고
  • 현재 CPU/메모리/IO 사용율
    • sar 1 100
    • vmstat 1 100
    • iostat 5 100
  • 페이징 스페이스 사용현황은?
    • lsps -s
  • 기술지원받을 때 FRU Number, Part Number를 물어볼 때 이 명령어를 사용하라.
    • lscfg -vl {장치식별자}
  • 현재 디스크 사용율
    • df -g : %Used가 90%이상이라면 로그를 지우거나 디스크를 증설하라
  • 시스템의 정보
    • 호스트명 : hostname
    • 접속자 id : id
    • 시스템 IP : ifconfig -a
    • OS정보 : oslevel -s , uname -a
  • 파일시스템 체크
    • fsck -y {마운트명} : 잘못된 부팅 후 체크용, 파일시스템이 고쳐질 때 잘못된 파일들은 사라질 수 있다!
  • 시스템 에러 메세지
    • errpt
    • errpt -aj {에러식별자}
  • 프로세스가 일정 수 이상 생기지 않는다면?
    • smit chgsys 에서 max process 의 수(Maximum number of PROCESSES allowed per user)를 조정
  • 핫 스왑으로 새로운 장치를 추가했는 데 사용할 수 없다. 목록에서 보이지 않는다.
    • cfgmgr

 

중요파일

  • /etc/inittab 파일 : 부팅시 초기화할 프로세스 목록, alog로 메세지를 알려준다.
    • init:2:initdefault:
      brc::sysinit:/sbin/rc.boot 3 >/dev/console 2>&1 # Phase 3 of system boot
      powerfail::powerfail:/etc/rc.powerfail 2>&1 | alog -tboot > /dev/console # Power Failure Detection
      powermig:2:wait:/etc/rc.powermig transition >/dev/null 2>&1 # powermig startup
      mkatmpvc:2:once:/usr/sbin/mkatmpvc >/dev/console 2>&1
      atmsvcd:2:once:/usr/sbin/atmsvcd >/dev/console 2>&1
      {ID}:{runlevel}:{action}:{command}
      {식별자}:{실행레벨}:{조치}:{명령어}
    • runlevel : 실행레벨
      • 0~9까지 존재
      • 2 : 기본, 다중 사용자 모드
      • 0 : 단일 사용자모드
      • 지정하지 않으면 모든 실행 레벨에서 실행됨
    • action : 조치
      • spawn : 프로세스가 시작되지 않았으면 시작
      • once : 프로세스를 시작, 중단되면 다시 시작
      • wait : 프로세스를 시작, 이 프로세스가 시작 완료가 되야 다음 프로세스가 시작됨
      • sysinit : 콘솔 사용전에 프로세스를 시작
    • 일반적으로 runlevel은 2에 action은 once 임
    • 명령어 > /dev/console 2>&1 : /dev/console 로 기본 출력하고 에러도 /dev/console로 출력하게 함
  • /etc/filesystems : 마운트 관런 파일 시스템 정보
    • /oracle: (마운트 위치)
              dev             = /dev/oralv
              vfs             = jfs2
              log             = /dev/loglv00
              mount           = true (부팅 시 자동마운트 설정, true 면 자동 마운트 함)
              options         = rw
              account         = false
  • /etc/hosts : hosts 파일
  • 증가하는 파일 관리
    • /var/adm/wtmp : 로그인, 시스템 관련 정보들
      • 사용법 목록보기 : who /var/adm/wtmp
    • /var/spool/*/* : 사용자별 메일, 크론탭, 프린트 등의 파일
      • 오래된 파일은 삭제
    • smit.log , smit.script
      • 너무 크다면 지우거나 줄여준다.
    • /etc/security/failedlogin : 로그인 실패 기록
      • 삭제 명령어 : cat /dev/null > /etc/security/failedlogin
    • /var/adm/sulog : su명령어의 목록

시스템 관리 명령어

  • SMIT : 시스템 관리 툴
    • 명령어 : smit , smitty
    • AIX의 약 70%의 시스템 제어가 가능
    • 특이점
      • 대화형 메뉴
      • ASCII 모드에서는 F1 같은 펑션키는 ESC+1 로 써야함
      • smit date 처럼 빠른패스를 사용할 수 있음
      • 로그
        • $HOME/smit.script : 스미티로 동작한 명령어
        • $HOME/smit.log :스미티 사용 내역
    • 사용팁
      • ESC+0 : 종료
      • ESC+3 : cancel(뒤로가기)
      • ESC+4 : 선택된 필드에서 사용할 수 있는 아이템 목록
      • ESC+7 : 다중 선택
      • ESC+8 : fast path
  • alog : 시스템 시작 메세지 보기 프로그램
    • 부팅 메세지 보기

    • 명령어 -s {서브시스템 명}
      • 명령어 -s {서브시스템}
      • 명령어 -p {PID}

LVM 간단 설명(사용금지)

  • LVM (Logical Volume Manager) (실제 사용할 일이 있을까??? 싶다.)(사용금지)
    • 실제 사용한다면 chXX, rmXX 관련해서는 꼭 알아보고 사용하라!
    • AIX의 LVM
      • 비연속공간사용, 여러개의 disk로 사용, 용량 추가 수월 등
    • 개념
      • 시스템에 사용할 수 있도록 하는 작업 순서
        1. VG에 PV(하드디스크)를 추가하여 용량을 지정하여 PP를 만든다.(smit mkvg)
        2. 원하는 만큼의 PP를 가지고 LV를 만든다.(smit lv)
        3. 파일시스템을 설정한다.(이때 마운트 위치도 설정한다) (smit jfs2)
        4. mount {마운트 위치}로 시스템에 마운트 시킨다.
