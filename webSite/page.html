웹 사이트 : 페이지 구분(기능,기술 등)

  • 페이지(page)
    • 정적(static) 페이지
      • 설명 : HTML 로만 이루어진 페이지
      • 구분
        • 빈(blank) 페이지
          • 설명 : 말 그대로 빈 페이지, 아무런 내용이 없다.
        • 컨텐츠(contents) 페이지
          • 설명 : 변화하지 않는 내용을 보여줄때 사용
          • 동작 : view
          • 내부구성 : 텍스트 , 이미지 , 동영상(swf)
    • 동적(dynamic) 페이지
      • 설명 : 쿼리스트링이나 기타 조건에 따라서 페이지가 변경되어 보여진다. 웹언어로 제작되어진 것(PHP , ASP , JSP , CGI)
      • 구분
        • 게시판(board)
          • 동작 : list , view , write(edit,answer) , delete
            • 권한에 따라서 사용하지 못하는 동작이 있을 수 있다.
            • 보통 DB를 사용한다.
            • list 동작시 search와 paging 기능이 추가적으로 사용된다.
          • DB스키마
            • 내용 : 글번호 , 게시판 아이디 , 작성자 , 아이디 , 비밀번호 , 제목 , 본문
            • 제어용 : 공지사항 , 비밀글 , HTML설정 , 삭제여부
            • 추가 내용 : 링크 , 조회수 , 첨부파일 , 리플 , 작성일, 수정일 , IP , 카테고리 , ETC필드
          • 스킨별
            • 게시판(일반형)
              • 단순하게 LIST , VIEW , WRITE 로 이루어진 스킨
            • 겔러리
              • 특이점 : answer 기능이 필요없다, list에 첨부된 이미지가 보인다. 썸네일 기능이 필요
            • 방명록
              • 특이점 : answer기능보다 reply 기능으로 답변하는 형식이 많다. 개인 아이콘등을 보여주는 경우가 많다.
            • 일정(달력)
              • 특이점 : 날짜에 대하여 보통 1달 단위로 보여준다. ETC 필드등을 사용
            • 특수 스킨
              • 요청에 따라서 추가적인 내용을 받거나 검색 조건에 제한하여 보여준다.
          • 서브 기능
            • 리플 (reply)
              • 동작 : list , view , write(edit,answer) , delete
              • DB스키마
                • 내용 : 글 번호 , 게시판 아이디 , 작성자 , 아이디 , 비밀번호 , 본문
                • 제어용 : HTML설정 , 삭제여부
                • 추가 내용 : 작성일, 수정일 , IP , 카테고리
            • 첨부파일(attach file)
              • 동작 : 게시판의 write(edit,answer) , delete에 연계하여 file upload
              • DB스키마
                • 내용 : 글 번호 , 첨부파일 명 , 첨부파일 위치
                • 제어 : 삭제여부
                • 추가 내용 : 이미지여부, 파일크기
            • 웹에디터(wysiwyg editer)
              • 용도 : 게시판의 write(edit,answer)에서 HTML을 실제 페이지로 보이면서 작성할 수 있도록 해준다.
              • 동작 : HTML 태그입력 , 이미지 업로드
        • 특수 프로그램
          • CMS : 컨텐츠 매니저먼트 시스템
            • 컨텐츠(contents) 페이지를 DB에 입력해놓고 조건에 따라서 해당 페이지가 컨텐츠 페이지 처럼 보이도록 해준다.
            • 게시판의 응용편
          • MENU : 메뉴관리
            • 동작 : 로그인별 , 메뉴 권한별,구분하여 메뉴를 보여주거나 안보여준다.
            • DB등으로 제어할 수도 있지만, 개인적으로 file를 ini형식으로 만들어 사용하는 걸 추천(메뉴등이 그렇게 자주 바뀌는게 아니므로)
          • MEMBER : 회원관리
            • 로그인 , 회원가입(수정,탈퇴) 등의 기능을 한다.
            • 게시판의 권한 제어에도 사용된다.
  • 기술
    • Client 측
      • javascript : 브라우저에서 보이는 HTML페이지를 제어하는데 사용된다.
      • AJAX : 백단으로 데이터 통신을 위해서 사용된다.(javascript의 일부분)
      • flash : 동적 컨텐츠용 , 메뉴와 네비게이션 부분에서 많이 사용된다.
    • Server 측
      • Web Server : Apache , IIS , Tomcat 등 (특정 포트(보통 80)의 HTTP프로토콜을 통한 요청에 응답을 하도록 한다.)
      • Web Language : PHP , ASP , JSP 등 (웹서버에선 지정된 확장자의 파일은 웹 언어로 처리하도록 하고 결과값을 받아서 출력한다)
      • DataBase : 데이터 관리에 사용
      • File Cache : 동적 페이지의 작성 리소스를 줄여준다.
      • FileSystem : NAS , NIS
  • 구조
    • HEADER
      • 로고
        • 내용 : 사이트 로고 이미지, 보통 클릭 가능하며 클릭하면 사이트 메인페이지로 이동된다.
      • 메뉴
        • TOP : TOP레벨의 메뉴가 보여진다. FLASH나 이미지로 처리하는 경우가 많다.
    • LEFT
      • 메뉴
        • LEFT : 선택된 TOP 레벨의 메뉴의 서브 메뉴가 나타난다. FLASH나 이미지로 처리하는 경우가 많다.
    • BODY(CONTENT)
      • 메뉴
        • NAVIGATION : 현재 페이지가 어느 메뉴인지 보여준다.
      • 컨텐츠
        • 정적(static) 페이지나 동적(dynamic) 페이지를 보여주는 부분
    • RIGHT
      • 메뉴
        • QUICK LINK : floating 레이어로 구성되는 경우가 많다. 바로가기 메뉴
    • FOOTER
      • BOTTOM : 회사소개, contact, 약관 , site map 등의 링크가 있다.
    • 레이아웃 예제
      • RIGHT가 없는 경우도 많다. 경우에 따라서 LEFT와 FOOTER가 없을 수도 있다.
      • HEADER
        메뉴 : TOP
        LEFT
        메뉴 : LEFT
        BODY
        메뉴 : NAVIGATION
        컨텐츠 : 동적 , 정적
        RIGHT
        메뉴 : QUICK LINK
        FOOTER
        메뉴 : BOTTOM
