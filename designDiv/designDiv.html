  • 디자인에서의 DIV
    • 이 문서는 HTML을 만들 때 DIV로 좀 더 간편하게 HTML문서를 디자인(제작)하는 것에 대한 짜리몽땅한 글이다.
  • DIV 사용이유
    • 소스의 간결성과 브라우저 렌더링 효율 증대 정도?
  • DIV의 TABLE 대체?
    • 적재적소라는 말이 있듯이 무조건 TABLE을 DIV로 바꾸는건 바보짓이다!
    • 간단한 모양이라면 DIV
    • GRID 형의 데이터 출력이라면 TABLE
  • 브라우저와 DTD에 따른 padding, margin , border 등의 처리 방법 차이
    • HTML 문서 디자인에서 padding, margin, border 는 엄청나게 중요하다.
      여기서 문제가 되는건 국내에서 점유율 90%(과장이 심하지만...)에 육박하는 IE가 문제이다.
    • IE(인터넷 익스플로러)가 왜 문제인가?
      • DTD를 사용하지 않을 경우 IE에서는 자체 DTD를 적용해서 padding과 margin 을 다른 브라우저와 다르게 처리한다.
      • DTD가 지정되지 않은 IE는
        • width에 border-width와 padding가 width속에 포함된다. 다른 브라우저는 포함되지 않는다.
      • 즉, DIV를 디자인으로 사용하기 위해서는 이 문제를 해결 후 사용해야한다.
        • 간단하게 HTML 4.01이나 XHTML 1.0 등의 DTD를 선언해주면 된다.
      • 참고 : 웹개발툴
